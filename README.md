# Cartograph

**Пассивная разведка поверхности атаки на графах с прозрачной моделью exposure-скоринга.**

Большинство recon-инструментов выдают *плоские списки* – тысячи поддоменов, IP и портов без структуры.
Cartograph вместо этого моделирует внешнюю поверхность атаки как **типизированный граф** (активы – узлы,
связи – рёбра) и накладывает поверх **документированный exposure-скоринг на признаках**. Цель не в том,
чтобы собрать больше данных, а в том, чтобы сделать собранное *связным*, *приоритизированным* и
*воспроизводимым*.

## Why a graph?

Плоский список скрывает как раз то, что важно аналитику:

- **Связность** – какие активы делят IP, ASN, сертификат или регистранта; где «мосты» между сегментами
  сети; какие узлы – точки концентрации риска.
- **Приоритизация** – обычно ~5% активов несут большую часть риска (забытые dev/staging-хосты, протухший
  TLS, wildcard-сертификаты, dangling-записи). Cartograph ранжирует активы по явному exposure-баллу, а не
  считает их равными.
- **Воспроизводимость** – результаты recon почти никогда не измеряют. Cartograph фиксирует входной снапшот,
  так что прогон детерминирован и измерим.

## Ethics & legality

Cartograph – **пассивный / OSINT-first** инструмент. Легальность встроена в архитектуру:

- **По умолчанию только пассивные публичные источники** – логи Certificate Transparency, passive DNS,
  WHOIS/RDAP, данные ASN/BGP, публичные веб-архивы. Никакого активного сканирования в дефолтном пути.
- **Активные проверки строго opt-in и за scope-guard** – опциональный лёгкий liveness-модуль (DNS-резолв,
  один HTTP HEAD/GET, чтение TLS) запускается *только* по явному флагу и *только* против целей из
  авторизованного allowlist (bug-bounty scope, который тебе разрешено тестировать, или твои домены).
  Никакой эксплуатации, брутфорса, фаззинга или нагрузки – никогда.
- **Scope-guard** – активные операции физически заблокированы вне подтверждённого allowlist.
- Встроенный троттлинг и дисковый кэш, чтобы уважать лимиты и ToS источников.

Ответственность за наличие авторизации на разведку любой цели лежит на тебе.

## Architecture

```
cartograph/
├── collectors/   # пассивные источники по цели: CT, RDAP, Wayback (единый интерфейс Collector)
├── enrich/       # обогатители графа над найденными активами: DoH/passive DNS (host->IP), ASN (IP->AS)
├── graph/        # типизированная модель узлов/рёбер + persistence (NetworkX; экспорт GraphML/Neo4j)
├── net.py        # общий async HTTP: клиент + кэш + троттлинг
├── pipeline.py   # двухфазный конвейер: сбор, затем обогащение
├── scoring/      # модель exposure: признаки F1-F7 + документированные веса + агрегатор
├── analysis/     # centrality, кластеры, takeover-кандидаты, snapshot diff
├── render/       # вывод CLI + отчёты; интерактивная HTML-визуализация
├── web/          # локальный веб-UI (FastAPI)
├── config.py     # scope-guard + конфигурация
└── cli.py        # точка входа (typer)
experiments/      # воспроизводимая эмпирическая оценка + снапшот данных
docs/             # methodology / whitepaper / дизайн-доки
tests/            # pytest
```

**Источники (пассивные, без ключей):** Certificate Transparency (crt.sh), RDAP (rdap.org), Wayback CDX
(web.archive.org), резолвинг через DNS-over-HTTPS (Cloudflare), ASN/BGP (RIPEstat / RIPE NCC). Mnemonic
passive DNS доступен через `--pdns` (его бесплатный тариф теперь отдаёт HTTP 402). DoH-запросы идут к
публичному резолверу, а не к цели, поэтому карта остаётся пассивной. Каждый источник кэшируется на диск,
так что повторный прогон по заполненному кэшу детерминирован и работает офлайн. Сбойные или лимитирующие
источники логируются и пропускаются – частичная карта всё равно сохраняется.

## Quickstart

```bash
git clone https://github.com/rrusmanov/Cartograph.git
cd cartograph
pip install -e ".[dev]"

# Пассивно построить карту поверхности атаки и ранжировать активы по exposure
cartograph collect shopify.com --score --out out/shopify.cartograph.json

# Пересчитать скоринг, проанализировать или сравнить сохранённый граф
cartograph score out/shopify.cartograph.json
cartograph analyze out/shopify.cartograph.json --takeover --clusters --centrality
cartograph diff out/last-week.json out/shopify.cartograph.json

# Построить интерактивный HTML-граф (хосты подсвечены по exposure)
cartograph visualize out/shopify.cartograph.json -o out/shopify.html
```

## Local web UI

```bash
pip install -e ".[web]"
cartograph serve                # http://127.0.0.1:8666
```

Вводишь домен в браузере и получаешь интерактивный граф, топ exposed-активов, takeover-кандидатов и
кластеры, с выгрузкой JSON/HTML. Сервер **локальный по замыслу** – он стреляет recon-запросами по
публичным источникам, поэтому биндится на `127.0.0.1` и не должен выставляться наружу.

Конвейер сбора пассивен (читает публичные базы; ни один пакет не доходит до цели). Активные
liveness-проверки – opt-in и за scope-guard.
